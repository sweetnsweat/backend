# EXP, 레벨, 골드 보상 정책

이 문서는 퀘스트 완료 보상, 사용자 레벨, 골드 지급 기준의 현재 백엔드 정책이다.

## 설계 기준

보상 기준은 `운동 시간 기반 노력량`을 1차 기준으로 둔다.

이유:

- 루틴 퀘스트는 이미 세션의 `estimatedMinutes`를 갖고 있어서 프론트와 백엔드가 같은 기준을 공유할 수 있다.
- MET나 칼로리는 운동 종류, 체중, 수행 강도에 따라 편차가 크다. 보상 기준으로 쓰면 특정 고강도 운동만 고르는 식의 파밍이 생길 수 있다.
- 발표/데모 기준에서는 사용자가 루틴을 만들고 완료했을 때 즉시 이해 가능한 보상 체계가 더 중요하다.
- 컨디션이 낮은 날에는 회복 퀘스트로 조정되므로, 강한 운동을 못 한 날에도 최소 보상을 줄 수 있어야 한다.

## 퀘스트 보상

### 루틴 퀘스트

루틴이 있는 날의 퀘스트는 활성 루틴의 오늘 세션을 완료하는 단위다.

```text
EXP = 세션 estimatedMinutes를 5 단위 반올림
최소 20 EXP
최대 60 EXP
Gold = EXP의 절반을 5 단위 반올림
최소 5 Gold
```

예시:

| 세션 예상 시간 | 지급 EXP | 지급 Gold |
| --- | ---: | ---: |
| 15분 | 20 | 10 |
| 30분 | 30 | 15 |
| 45분 | 45 | 25 |
| 70분 | 60 | 30 |

최소 20 EXP를 둔 이유는 짧은 초보자 루틴도 완료 경험을 보상하기 위해서다. 최대 60 EXP를 둔 이유는 지나치게 긴 루틴 하나로 보상을 과하게 얻는 것을 막기 위해서다.

### 오프데이 퀘스트

루틴이 없는 날에는 걷기/스트레칭 같은 짧은 회복 활동을 기준으로 한다.

```text
EXP = 목표 분을 5 단위 반올림
최소 10 EXP
최대 25 EXP
Gold = EXP의 절반을 5 단위 반올림
최소 5 Gold
```

오프데이는 루틴 수행일보다 낮은 보상을 준다. 대신 사용자가 스토리 진행 중 퀘스트를 만나도 빈 응답이 되지 않게 하는 완충 역할이다.

### 회복 퀘스트

컨디션이 낮은 날에는 루틴 대신 회복 퀘스트가 생성된다.

```text
EXP = 10
Gold = 5
```

회복 퀘스트는 사용자의 컨디션을 고려한 최소 보상이다. 운동 강도를 낮추는 대신, 앱 사용 흐름이 끊기지 않도록 한다.

## 레벨 정책

레벨은 누적 EXP 기준이다.

```text
레벨 N 도달에 필요한 누적 EXP = 50 * N * (N - 1)
```

구간별 필요 EXP는 현재 레벨에 비례해 증가한다.

```text
Lv.1 -> Lv.2: 100 EXP
Lv.2 -> Lv.3: 200 EXP
Lv.3 -> Lv.4: 300 EXP
Lv.4 -> Lv.5: 400 EXP
```

주요 레벨 기준:

| 레벨 | 필요 누적 EXP |
| --- | ---: |
| 1 | 0 |
| 2 | 100 |
| 3 | 300 |
| 4 | 600 |
| 5 | 1,000 |
| 10 | 4,500 |
| 20 | 19,000 |

이 곡선을 선택한 이유:

- 초반에는 30분 루틴 3~4회 정도로 Lv.2에 도달해 성취감이 빠르게 온다.
- 레벨이 올라갈수록 다음 레벨 요구량이 선형으로 증가해서 장기 사용 동기가 유지된다.
- 계산식이 단순해서 백엔드, DB 마이그레이션, 프론트 표시가 모두 같은 기준을 쓰기 쉽다.

## 지급 안정성

퀘스트 완료 보상은 중복 지급되지 않는다.

- EXP는 `user_exp_logs(user_id, ref_type, ref_id)` 기준으로 한 퀘스트당 한 번만 기록한다.
- Gold는 `wallet_transactions(user_id, tx_type, ref_type, ref_id)` 기준으로 한 퀘스트당 한 번만 기록한다.
- 사용자가 같은 완료 API를 여러 번 호출해도 `users.total_exp`, `users.level`, `wallets.balance_currency`가 중복 증가하지 않는다.

## 퀘스트 응답 필드

오늘 퀘스트 조회, AI 서버용 오늘 퀘스트 조회, 퀘스트 완료 응답에는 아래 보상 필드가 포함된다.

```json
{
  "rewardExp": 30,
  "rewardGold": 15,
  "rewardCurrency": 15
}
```

표시 기준:

- `rewardExp`: 완료 시 지급될 EXP
- `rewardGold`: 완료 시 지급될 Gold. 프론트 표시는 이 필드를 사용한다.
- `rewardCurrency`: 기존 호환용 재화 필드. 현재는 `rewardGold`와 같은 값이다.

## 프론트 표시 필드

`GET /api/users/me`, 로그인 응답, 온보딩 저장 응답에서 아래 필드를 사용한다.

```json
{
  "level": 3,
  "totalExp": 420,
  "currentLevelExp": 120,
  "nextLevelRequiredExp": 300,
  "nextLevelRemainingExp": 180,
  "balanceCurrency": 65
}
```

표시 기준:

- `level`: 현재 레벨
- `totalExp`: 전체 누적 EXP
- `currentLevelExp`: 현재 레벨 안에서 쌓은 EXP
- `nextLevelRequiredExp`: 다음 레벨까지 필요한 전체 구간 EXP
- `nextLevelRemainingExp`: 다음 레벨까지 남은 EXP
- `balanceCurrency`: 현재 보유 Gold
